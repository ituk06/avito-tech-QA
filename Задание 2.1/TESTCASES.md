# Тест-кейсы для API объявлений (просто и понятно)

Это мои тесты для API сервиса объявлений. Я проверял разные ситуации, когда всё работает хорошо и когда что-то идет не так.

**Базовый адрес сервиса:** https://qa-internship.avito.com

## 1. Сохраняем объявление

### Хорошие тесты
**TC001: Сохраняем объявление правильно**
*Что делаем:*
- Отправляем POST запрос на `BaseUrl/api/1/item`
- В теле пишем:
```json
{
   "sellerID": 4962657,
   "name": "test",
   "price": 500,
   "statistics": {
     "likes": 100,
     "viewCount": 100,
     "contacts": 100
  }
}
```
*Что ждем:*
- Статус 200 (всё ок)
- В ответе будет:
```json
{
   "id": "какой-то-уникальный-номер",
   "sellerId": 4962657,
   "name": "test",
   "price": 500,
   "statistics": {
     "likes": 100,
     "viewCount": 100,
     "contacts": 100
  },
   "createdAt": "дата-и-время-создания"
}
```

### Плохие тесты
**TC002: Пробуем сохранить без обязательных полей**
*Что делаем:*
- Отправляем POST запрос на `BaseUrl/api/1/item`
- В теле намеренно не пишем одно из обязательных полей (например, price)
*Что ждем:*
- Статус 400 (ошибка)
- Сообщение, что пропущено обязательное поле

**TC003: Пробуем сохранить с неправильными данными**
*Что делаем:*
- Отправляем POST запрос на `BaseUrl/api/1/item`
- В теле делаем тип поля неправильным, например:
```json
{
   "sellerId": "4962657",  // строка вместо числа
   "name": "test",
   "price": 1000,
   "statistics": {
     "likes": 5,
     "viewCount": 150,
     "contacts": 10
    }
}
```
*Что ждем:*
- Статус 400 (ошибка)
- Сообщение, что данные не подходят

## 2. Получаем объявление по id

### Хорошие тесты
**TC004: Получаем реально существующее объявление**
*Что делаем:*
- Сначала создаем тестовое объявление (чтобы взять его id)
- Потом отправляем GET запрос на `BaseUrl/api/1/item/{id}`
*Что ждем:*
- Статус 200 (всё ок)
- В ответе — все данные объявления в правильном формате

### Плохие тесты
**TC005: Пытаемся получить несуществующее объявление**
*Что делаем:*
- Отправляем GET запрос на `BaseUrl/api/1/item/{придуманный-id}`
*Что ждем:*
- Статус 404 (не найдено)
- Сообщение, что такого объявления нет

**TC006: Пытаемся получить объявление с неправильным форматом id**
*Что делаем:*
- Отправляем GET запрос на `BaseUrl/api/1/item/абракадабра`
*Что ждем:*
- Статус 400 (ошибка)
- Сообщение, что id должен быть в правильном формате

## 3. Получаем все объявления пользователя

### Хорошие тесты
**TC007: Получаем объявления существующего пользователя**
*Что делаем:*
- Сначала создаем объявление для какого-то пользователя
- Отправляем GET запрос на `BaseUrl/api/1/{sellerID}/item`
*Что ждем:*
- Статус 200 (всё ок)
- В ответе — массив с объявлениями
- Все объявления в массиве принадлежат именно этому пользователю

**TC008: Получаем объявления пользователя, у которого их нет**
*Что делаем:*
- Знаем id пользователя, у которого нет объявлений
- Отправляем GET запрос на `BaseUrl/api/1/{sellerID}/item`
*Что ждем:*
- Статус 200 (всё ок, просто пусто)
- В ответе — пустой массив []

### Плохие тесты
**TC009: Пытаемся получить объявления несуществующего пользователя**
*Что делаем:*
- Отправляем GET запрос на `BaseUrl/api/1/{придуманный-id-пользователя}/item`
*Что ждем:*
- Статус 404 (не найдено)
- Сообщение, что такого пользователя нет

**TC010: Пытаемся получить объявления с неправильным форматом sellerID**
*Что делаем:*
- Отправляем GET запрос на `BaseUrl/api/1/абракадабра/item`
*Что ждем:*
- Статус 400 (ошибка)
- Сообщение, что sellerID должен быть числом
